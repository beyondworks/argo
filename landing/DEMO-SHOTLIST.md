# Argo 랜딩 — 데모 영상 샷리스트

랜딩의 각 기능 셀(16:9 플레이스홀더) 자리에 들어갈 데모 영상 목록. 총 **13컷**.
현재 전부 정적 플레이스홀더(재생 아이콘)로 표시 중이며, 아래 영상이 준비되면 그 자리에 스왑된다.

## 공통 스펙 (전 컷 동일)

- **화면비**: 16:9 고정 (셀이 `aspect-ratio: 16/9`)
- **해상도**: 1920×1080 이상으로 녹화 → 최소 1280×720 납품
- **길이**: 아래 각 컷의 "길이"는 권장값(6~20초). 시작·끝 상태를 깔끔하게 맞춰 **루프가 튀지 않게** (muted·autoplay·loop로 재생됨)
- **소리**: 불필요 (무음 재생)
- **무드**: 랜딩이 순수 블랙 배경이라 **다크 테마 UI**로 녹화하면 가장 잘 붙음. 커서는 보이되 움직임은 느리고 부드럽게
- **화면 정리**: 알림·개인정보·실명 토큰/키 노출 금지. 데모용 워크스페이스로 깨끗한 상태에서 촬영
- **한 컷 = 한 메시지**: 그 기능의 핵심 동작 하나만 명확히 증명. 잡다한 클릭 최소화
- **납품 형식**: mp4로 주시면 됨 → 내가 히어로 영상처럼 ffmpeg로 web용(webm/mp4 loop)으로 변환·연결

## 납품 후 연결

각 영상은 `landing/public/assets/demos/featN.webm`(또는 .mp4)로 배치 →
`components/FeatureBlock.jsx`의 `DemoPlaceholder`에 `video` prop만 넘기면 정적 셀이 영상으로 바뀐다.
파일명은 아래 표의 **파일명**을 그대로 쓰면 코드 매핑이 1:1로 맞는다.

## 1장 · 에이전트 생성 (Create Agents)

### 컷 01 — feat1 · "직무를 말하면, 에이전트가 만들어집니다"
- **파일명**: `feat1.webm` · **길이** ~12s
- **증명할 것**: 프롬프트 한 줄로 역할·규칙·도구 갖춘 에이전트가 즉시 생성됨
- **촬영**: 입력창에 `"마케팅 카피 쓰는 직원 뽑아줘"` 한 줄 타이핑 → 엔터 → 에이전트 카드가 이름·역할·도구와 함께 생성되는 순간까지. 타이핑은 실제 타자감 살려서

### 컷 02 — feat2 · "에이전트들이 서로 협업합니다"
- **파일명**: `feat2.webm` · **길이** ~18s
- **증명할 것**: 에이전트끼리 일을 나누고 넘기고 검토 (사람 개입 없음)
- **촬영**: 2명 이상 에이전트가 있는 워크스페이스에서, 한 에이전트가 다른 에이전트에게 작업을 넘기고 → 응답이 오가는 흐름. 메시지 주고받는 라인이 보이게

## 2장 · 무한 기억 (Memory)

### 컷 03 — feat3 · "설치 한 번에 풀 패키지"
- **파일명**: `feat3.webm` · **길이** ~15s
- **증명할 것**: 앱 설치만으로 장기기억·규칙·하네스가 로컬에 세팅
- **촬영**: 설치 완료 → 첫 실행 시 로컬에 기억/규칙/하네스 폴더 구조가 갖춰지는 화면 (설치 진행바 또는 `~/.argo` 같은 폴더가 채워지는 장면)

### 컷 04 — feat4 · "마크다운 한 장이 아니라, 폴더째 기억합니다"
- **파일명**: `feat4.webm` · **길이** ~11s
- **증명할 것**: 기억이 단일 파일이 아니라 폴더 전체
- **촬영**: 기억 디렉토리를 열어 여러 파일·하위 폴더가 트리로 펼쳐지는 모습. "한 장 vs 폴더째" 대비가 보이게

### 컷 05 — feat5 · "LLM 위키 — 기억이 서로 연결됩니다"
- **파일명**: `feat5.webm` · **길이** ~20s
- **증명할 것**: 기억이 위키처럼 링크되고 에이전트가 공유 (지식 복리)
- **촬영**: 위키 뷰에서 노트 간 링크를 타고 이동하거나, 연결 그래프가 그려지는 장면. **※ 제품에 위키 UI가 구현돼 있으면 실녹화, 아니면 목업 필요 — 구현 상태 확인 요망**

### 컷 06 — feat6 · "필요한 기억만 읽어 토큰을 아낍니다"
- **파일명**: `feat6.webm` · **길이** ~14s
- **증명할 것**: 키워드 검색으로 관련 기억만 골라 컨텍스트에 투입
- **촬영**: 검색창에 키워드 입력 → 관련 노트만 필터링되어 뜨고 → 그게 답변 컨텍스트로 들어가는 흐름. 가능하면 "전체 대비 일부만 로드" 느낌 강조

## 3장 · 이어지는 맥락 (Continuity)

### 컷 07 — feat7 · "텔레그램, 클릭 한 번에 연결"
- **파일명**: `feat7.webm` · **길이** ~9s
- **증명할 것**: 복잡한 설정 없이 원클릭 연결
- **촬영**: "텔레그램 연결" 버튼 클릭 → QR/딥링크 → 연결 완료 상태 배지까지. 짧고 경쾌하게

### 컷 08 — feat8 · "PC에서 하던 대화, 텔레그램으로 이어서"
- **파일명**: `feat8.webm` · **길이** ~13s
- **증명할 것**: 책상 대화를 이동 중 텔레그램에서 그대로 이어감 (맥락 유지)
- **촬영**: 좌: PC에서 대화 중 → 우: 휴대폰 텔레그램에 같은 스레드가 이어지는 분할/전환. **화면(PC) + 실기기(폰) 동시 촬영 또는 합성 필요**

### 컷 09 — feat13 · "기기가 바뀌어도, 로그인하면 그대로"
- **파일명**: `feat13.webm` · **길이** ~14s
- **증명할 것**: 새 기기에서 로그인 한 번으로 기억·규칙·진행 중 일 복원
- **촬영**: 빈 새 컴퓨터(또는 새 프로필) → 로그인 → 기억·규칙·진행 중이던 작업이 동기화되어 채워지는 장면

## 4장 · 스킬 자동화 (Skills)

### 컷 10 — feat9 · "2번 반복되면, 자동으로 스킬이 됩니다"
- **파일명**: `feat9.webm` · **길이** ~16s
- **증명할 것**: 반복 업무를 에이전트가 알아서 스킬로 저장
- **촬영**: 같은 작업을 1회 수행 → 2회째 수행 → "스킬 생성됨" 토스트/항목이 스킬 목록에 추가되는 순간. 반복→저장 인과가 보이게

### 컷 11 — feat10 · "스킬·MCP 전부 원클릭 설치"
- **파일명**: `feat10.webm` · **길이** ~10s
- **증명할 것**: Skillsmp의 스킬·MCP를 클릭 한 번에 설치
- **촬영**: Skillsmp 마켓 화면 → 스킬 하나 "설치" 클릭 → 설치 완료되어 에이전트 능력에 추가되는 흐름

## 5장 · 자동 실행 (Automation)

### 컷 12 — feat11 · "예약해두면, 아침에 완성되어 있습니다"
- **파일명**: `feat11.webm` · **길이** ~17s
- **증명할 것**: 작업 예약 → 정해진 시각 자동 실행 → 결과 보고
- **촬영**: 스케줄러에서 반복 작업 예약(시각 설정) → 예약됨 상태 → (선택) 실행되어 결과가 쌓인 뷰. 예약 UI가 핵심

### 컷 13 — feat12 · "대기 중엔 토큰 0원"
- **파일명**: `feat12.webm` · **길이** ~8s
- **증명할 것**: 에이전트가 놀 때 비용 0
- **촬영**: 대기 상태에서 사용량/비용 미터가 **0**에 머무는 모습. 짧게, 숫자 0이 명확히 보이게

## 우선순위 제안

정식 오픈까지 13컷을 한 번에 못 찍는다면, **Core Four(최우선 강조 4개)를 증명하는 컷 먼저**:
1. **feat5** (LLM 위키) — Core Four ②
2. **feat13** (기기 동기화) — Core Four ③
3. **feat8** (PC→텔레그램 이어가기) — Core Four ④
4. **feat4** (폴더째 기억) — Core Four ① 무한 기억의 시각 증거
그다음 feat1(에이전트 생성)·feat7(텔레그램 연결)이 첫인상 컷으로 임팩트 큼.

## 촬영 불가/목업 필요 후보 (구현 상태 확인 요망)

feat5(위키 UI)·feat8(대화 이어가기)·feat13(기기 동기화)·feat9(자동 스킬화)는 제품에 해당 UI가 있어야 실녹화가 됩니다. 아직 미구현이면 (a) 목업 화면으로 촬영하거나 (b) 해당 컷은 후순위로 미루고 정적 플레이스홀더 유지 — 어느 쪽으로 갈지는 유건님이 구현 상태 보고 정해주세요.
